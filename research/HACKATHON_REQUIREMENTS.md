## Hackathon: HackerRank Orchestrate (June 2026)
**Topic**: Multi-Modal Evidence Review  
**Duration**: 24 hours  
**Challenge**: Build a system that verifies damage claims using submitted images, a short claim conversation, user history, and minimum evidence requirements.

## What We Built Beyond Requirements

### 1. Ensemble Architecture
- **YOLO11-nano**: Automatic damage detection + localization
- **MobileNetV3-Large**: Multi-task classification (object, issue, part, damage)
- **Qwen 2.5-VL-7B**: Visual perception with structured JSON output
- **Cross-check logic**: Confidence-based arbitration between models

### 2. Label Normalization
- Embedding-based mapping from VLM descriptions to canonical labels
- Uses `intfloat/multilingual-e5-small` for similarity computation
- Handles synonyms (e.g., "windshield" = "wind shield")

### 3. Advanced Training
- **YOLO**: 150 epochs, AdamW, cosine LR, mosaic augmentation
- **MobileNet**: Transfer learning from ImageNet, 30 epochs, early stopping
- **Data augmentation**: Light (RandomHorizontalFlip + ColorJitter) proved better than heavy

### 4. Rule Engine
- 7 adjudication rules with confidence thresholds
- Wrong object → contradicted (not NEI)
- Severity exaggeration detection (12+ pattern matching)
- User history risk integration

### 5. Infrastructure
- **SLURM jobs**: Batch training and inference on Tartu HPC
- **GitHub integration**: CI/CD via commits and pushes
- **Testing**: 86 tests covering all components
- **Logging**: Comprehensive per-claim logging with reasoning

## What We Did NOT Build (and why)

| Feature | Reason |
|--------|--------|
| GPT-4V/Claude API integration | Cost (~$0.01/image); Qwen is free and open-source |
| Real-time streaming | Not required; batch processing is sufficient |
| Web UI | Not required; CLI + CSV output is evaluable |
| Model distillation | Time constraint; 24-hour hackathon |
| Active learning loop | No human-in-the-loop feedback mechanism provided |
| Multi-GPU training | Single V100 is sufficient for small models |

## Evaluation Strategy

### How We Tested
1. **Unit tests**: 86 tests covering all components (config, data loading, inference, normalization, integration, MobileNet, three-stage pipeline)
2. **Sample evaluation**: Run inference on `sample_claims.csv` and compare against expected behavior
3. **Cross-validation**: MobileNet trained on 2818 images, validated on 604 images
4. **YOLO validation**: mAP50 0.448 on 772 validation images

### Metrics We Track
- **Issue type accuracy**: ~80-82% (MobileNet validation)
- **Object part accuracy**: ~54-59% (MobileNet validation)
- **Damage detection accuracy**: ~95-97% (MobileNet validation)
- **YOLO mAP50**: 0.448 (damage detection + classification)

### Known Limitations
- Test set has no ground truth labels (44 rows) - cannot compute accuracy locally
- Sample set is small (20 rows) - limited statistical power
- VLM inference is slow (~15-20s per claim) - acceptable for batch processing
- OOM on 16GB GPU requires 32GB GPU or memory optimization

## Summary

We built a **production-ready, auditable, multi-modal evidence review system** that:
- ✅ Meets all hackathon requirements
- ✅ Uses open-source models (zero API cost)
- ✅ Has deterministic, traceable decisions
- ✅ Includes comprehensive testing and evaluation
- ✅ Is modular and extensible
- ✅ Processes 44 claims in ~15-20 minutes on a single GPU

**Key differentiator**: The three-stage separation (perception → evidence → adjudication) makes the system robust, auditable, and easy to maintain - unlike end-to-end LLM approaches that are black boxes.
